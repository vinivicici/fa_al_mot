# 전처리 과정에서 버려지는 정보 분석

## 1. H&M 데이터 전처리에서 버려지는 칼럼들

### 1.1 hnm_column_drop.py에서 제거되는 칼럼들 (17개)

#### 제품 식별 관련
- `prod_name`: 제품명 (의미 없는 값으로 판단)
- `article_id`: 개별 상품 ID (product_code로 통합)
- `product_type_no`: 제품 타입 번호

#### 색상 관련
- `colour_group_code`: 색상 그룹 코드
- `colour_group_name`: 색상 그룹명
- `perceived_colour_value_id`: 인지된 색상 값 ID
- `perceived_colour_value_name`: 인지된 색상 값명
- `perceived_colour_master_id`: 인지된 색상 마스터 ID
- `perceived_colour_master_name`: 인지된 색상 마스터명

#### 그래픽/디자인 관련
- `graphical_appearance_no`: 그래픽 외관 번호
- `graphical_appearance_name`: 그래픽 외관명 (이미지 벡터화 시 불필요)

#### 조직 구조 관련
- `department_no`: 부서 번호
- `department_name`: 부서명
- `index_code`: 인덱스 코드
- `index_name`: 인덱스명
- `index_group_no`: 인덱스 그룹 번호
- `section_no`: 섹션 번호
- `garment_group_no`: 의류 그룹 번호

#### 하위 카테고리
- `section_name`: 섹션명 (속옷, 악세서리 등)
- `product_type_name`: 제품 타입명 (중분류인 product_group_name으로 대체)

**제거 이유**: 
- 같은 제품의 다른 색상은 같은 가격을 가짐 (색상은 가격에 영향 없음)
- 중분류만 남기고 소분류는 제거 (다른 데이터셋과 통합을 위함)
- 이미지가 벡터화되므로 그래픽 정보 불필요

## 주요 버려지는 정보 요약

### H&M
1. **색상 관련 정보**: 같은 제품 다른 색상은 같은 가격이므로 불필요
2. **상세 분류 정보**: 중분류만 남기고 소분류 제거
3. **그래픽 정보**: 이미지 벡터화 시 불필요
4. **식별자**: 개별 ID 대신 product_code 사용

### Fashion
1. **이미지 URL들**: images, style360Images, styleImages 전체 버려짐
2. **상세 속성 정보**: articleAttributes (재질, 핏, 길이 등 368개 필드)
3. **색상 정보**: baseColour, colour1, colour2
4. **시즌/연도**: season, year
5. **상품 표시 정보**: variantName, articleNumber, displayCategories, productDisplayName
6. **메타데이터**: meta.code, meta.requestId
7. **상업적 정보**: 할부, 현금배송, 평점 등

**버려지는 이유:**
- 이미지 URL은 별도 이미지 데이터셋에서 관리
- 상세 속성은 너무 세분화되어 카테고리화가 어려움
- 색상은 가격에 영향 없음
- 시즌/연도는 분석에 직접 사용하지 않음
- 메타데이터는 전처리에 불필요
